---
description: プラン作成から全タスク実行までを1コマンドで行う(実行前に確認あり)
---

プランを作成し、内容の確認を経て、全タスクを実行してください。種類: $ARGUMENTS(refactor / review / modernize、または自由指示。省略時は refactor)

進め方は既存スキルの組み合わせです。独自の手順を発明しないこと:

## 1. プランを書く

`skills/write-plan/SKILL.md` に厳密に従い、リポジトリを実際に調査してフォーマット契約どおりに `docs/plans/<type>/PLAN.md` を作成する。この段階では実装しない。

## 2. 実行前に一度停止する

完成したプランの要約(フェーズ構成・タスク数・特にリスクの高いタスク)と PLAN.md のパスを提示し、**ユーザーの GO を待つ**。書き立てのプランは前提の誤りを含みうるため、この停止は省略しないこと。修正指示があればプランを直し、再度確認する。

## 3. 全タスクを実行する

GO が出たら、`skills/execute-plan/SKILL.md` の手順で未チェックタスクを上から順に**すべて**実行する(実装 → 検証 → `- [x]` に更新 + 進捗ログ追記)。連続実行の規律は `/plan-all` と同じ:

- 検証はタスクごとに実行する。まとめて最後に1回、にしない
- 検証失敗が直せない・タスクがブロック・プラン自体の誤りを発見、のときはそこで停止して報告する

完了したら(または停止したら)、実行したタスク一覧・各検証結果・残タスクを要約して報告する。進捗は `agent-ready plan status` でも確認できます。

途中レビューを挟みたい長いプランでは、このコマンドではなく `/plan-write` → `/plan-next` + タスクごとのコミットを使ってください。
