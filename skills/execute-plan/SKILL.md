# プラン実行

## いつ使うか

agent-ready の `plan create` が生成したプラン(`docs/plans/<type>/PLAN.md`)を、1タスクずつ確実に消化するとき。どのAIエージェント・モデルからでも同じ手順で使えます。

## 手順

1. 対象プランを読む。指定がなければ `docs/plans/` 配下で未チェックタスクが残っている最初の `PLAN.md` を使う。
2. **最初の未チェックタスク(`- [ ]`)を1つだけ**選ぶ。フェーズは上から順に消化し、タスクに依存関係の注記があれば従う。
3. タスクに書かれた対象ファイル・手順に従って実装する。作業種別に応じて `skills/refactor` や `skills/bug-fix` の手順書にも従う。
4. タスクに書かれた検証を実行する。
5. 検証が通ったら、そのタスクのチェックボックスを `- [x]` に更新し、PLAN.md 末尾の「進捗ログ」に1行追記する(タスクID・変更概要・検証結果)。
6. そこで**停止**し、実施内容を報告する。次のタスクには進まない。

## ルール

- 1回の実行で1タスクのみ。まとめて消化しない(1タスク = 1コミット = レビュー可能な単位を保つため)。
- 検証が通っていないタスクにチェックを付けない。ブロックされた場合はチェックを付けず、進捗ログに理由を記録して停止する。
- **タスクを完了せずに停止するときは、理由が何であれ進捗ログに `(中断: <タスクID> — <理由>)` を1行残してから報告する。** チャットの報告はセッションと一緒に消えるが、PLAN.md は残る — 後からプランを見ただけで「どこで・なぜ止まったか」が分かる状態を保つ。
- プラン自体に誤りや前提のずれを見つけたら、タスクを実行せずプランの修正案を報告する。
- 挙動を変えないタスク(リファクタリング)で挙動変更が必要になったら、それは別タスクとして提案する。

## 連続実行(/plan-all)

すべてのタスクを一気に消化するよう指示された場合(Claude Code では `/plan-all`)も、手順 1〜5 はタスクごとにそのまま実行する(検証・チェック更新・進捗ログを1タスクずつ)。変わるのは手順 6 だけで、停止せず次の未チェックタスクへ進む。検証失敗・ブロック・プランの誤りを見つけたら、その時点で停止して報告する(このときも進捗ログに中断行を残す)。タスクが大きく1ターンで完走しきれない恐れがあるプランでは、`/plan-all` ではなくタスクごとにターンを仕切り直す `/loop /plan-next` が向いている。
