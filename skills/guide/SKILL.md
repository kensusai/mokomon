# 使い方ガイド(入口案内)

## いつ使うか

- どんなスキル・コマンドがあるか知りたいとき。
- やりたいことはあるが、どの入口を使えばいいか分からないとき。

## 手順

1. **実在するものだけを案内する。** `.claude/commands/` と `skills/` を一覧し、案内対象の frontmatter の description を読む。このプロジェクトに生成されているものが正であり、この手順書の分類例より優先する。
2. **やりたいことを把握する。** 指定があればそれを使い、なければ「いま何をしたいか」を1問だけ聞く。「何があるか知りたい」だけなら、カテゴリ別一覧(下の分類)を各1行の説明付きで提示して終了する。
3. **入口を1つ選んで案内する。** 「なぜそれか」「実行すると何が起きるか」「どう打つか(コマンド例)」の3点で示す。候補が複数あるなら推奨1つ+代替を1行で。
4. **望まれたらそのまま実行に移る**(該当スキルの手順に従う)。

## 分類の目安

- **作る**: `/add-feature`(機能追加)、`/design`(デザインモック)、`/greenfield`(新規開発の司令塔 — 工程に迷ったら)、ワークフローフェーズの各コマンド
- **直す**: `/fix-bug`(バグ修正)、`/perf`(性能問題)
- **良くする**: `/review`(いまの変更の差分レビュー)、`/codebase-review`(コード全体の棚卸し → 指摘一覧)、`skills/refactor`(挙動を変えない整理)
- **計画して進める**: `/plan-run`(計画 → 確認 → 全実行)、`/plan-write` / `/plan-next` / `/plan-all`(個別に)
- **保守**: `/test`(テスト実行と修正)、`skills/docs-update`(ドキュメント同期)

## 必ず伝えること

- **コマンドを覚える必要はない。**やりたいことを普通の言葉で頼めば、エージェントが該当スキルを自動で使う。コマンドは「確実にその手順で動かしたいとき」のショートカット。
- 一覧は `docs/ai-workflow.md` の対応表にもある。

## ルール

- 存在しないコマンド・スキルを案内しない(必ず手順1で確認してから)。
- 一度に全部説明しない。聞かれたことに必要な分だけ答える。
